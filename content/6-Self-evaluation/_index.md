During my internship at **Amazon Web Services Vietnam Company Limited** from **May 5, 2026** to **July 30, 2026**, I had the opportunity to work in a professional environment, learn about the process of deploying solutions on cloud computing platforms, and apply the knowledge I had acquired to a practical project.

I participated in the **Workforce Bootcamp - First Cloud AI Journey** program, in which the learning content focused on foundational AWS services and Cloud system architecture design methods. During the initial stage, I studied Cloud Computing, AWS Global Infrastructure, Amazon EC2, Amazon S3, Amazon VPC, AWS IAM, Amazon DynamoDB, AWS Lambda, Amazon API Gateway, and Amazon Cognito.

Based on this foundational knowledge, I continued to research and implement the project:

**“Serverless & Spot Instance Backend Architecture for Live-Service Games on AWS”**

During the project implementation, I participated in requirement analysis, system architecture design, data model development, and the deployment of Backend components for a multiplayer game. The solution was designed by combining Serverless architecture with Amazon EC2 Spot Instances to meet requirements for scalability, real-time processing, security, and operational cost optimization.

Throughout the internship, I not only strengthened my professional knowledge of AWS but also improved my ability to analyze requirements, design systems, independently research technical documentation, and troubleshoot issues. I gained a clearer understanding that building a Cloud system requires more than implementing the correct functionality; it also requires the simultaneous consideration of multiple factors such as performance, security, availability, scalability, and operational cost.

Regarding my working style, I always made an effort to complete labs, assignments, and report content according to the required schedule. When encountering issues during implementation, I proactively reviewed documentation, checked logs, tested multiple approaches, and discussed with other members to identify the cause. I also seriously considered feedback from my supervisor in order to adjust my implementation approach and improve the quality of my work.

In addition to technical knowledge, the internship helped me develop teamwork, idea presentation, progress tracking, and structured documentation skills. Participating in activities, events, and sharing sessions within the program also broadened my perspective on careers in Cloud Computing and Cloud Architecture.

### Areas for Improvement

* I need to continue improving my ability to transform theoretical knowledge into complete architecture designs for large-scale systems with multiple dependent components.

* I need to improve my ability to identify the root causes of issues instead of focusing only on their visible symptoms, especially for problems related to WebSocket, distributed state, and asynchronous processing flows.

* I need to gain more experience in load testing so that I can accurately evaluate the capacity of the Matchmaker, Lambda, DynamoDB, and Game Server when the number of concurrent users increases significantly.

* I need to study Infrastructure as Code in greater depth, particularly AWS CloudFormation, AWS CDK, or Terraform, so that infrastructure deployment can be standardized, repeatable, and better version-controlled.

* I need to improve my knowledge of Cloud security, including designing IAM Policies according to the principle of least privilege, managing secrets, and protecting public APIs.

* I need to improve my work scope management skills, prioritize important functions, and allocate time appropriately among research, implementation, testing, and documentation.

* I need to be more proactive in presenting design options, analyzing their advantages and disadvantages, and defending technical decisions before the team or supervisor.

* I need to maintain the habit of writing technical documentation throughout the implementation process instead of concentrating on completing documentation only during the final stage.

### Development Direction After the Internship

After the internship, I plan to continue developing in the fields of **Cloud Computing**, **Cloud Backend**, and **Cloud Architecture**. The topics I intend to continue studying include:

* Studying the AWS Well-Architected Framework in greater depth.
* Practicing Infrastructure as Code with Terraform or AWS CDK.
* Researching Auto Scaling, Load Balancing, and multi-Availability Zone architectures.
* Studying Event-Driven Architecture and Distributed Systems in greater depth.
* Practicing load testing, monitoring, and Cloud cost optimization.
* Researching CI/CD and GitOps processes for applications on AWS.
* Continuing to improve the project so that it can be deployed and tested at a larger scale.
