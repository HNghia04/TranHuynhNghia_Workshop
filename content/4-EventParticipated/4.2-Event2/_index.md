# Summary Report: Cloud Architect

### Event Information:

* **Event Name:** Cloud Architect
* **Event Date:** June 19, 2026
* **Event Format:** In-person team competition
* **Organizer:** First Cloud AI Journey (FCAJ) – Amazon Web Services Vietnam
* **Participants:** Interns from the First Cloud AI Journey program

---

## Purpose of the Event

* **Assess Cloud Computing Knowledge:** Evaluate participants' ability to apply AWS knowledge and design system architectures through a team-based competition.

* **Develop Cloud Architecture Thinking:** Help participants improve their ability to select AWS services that are appropriate for specific real-world problems instead of merely memorizing theoretical concepts.

* **Enhance Teamwork Skills:** Strengthen the ability to discuss, analyze, and make decisions within a limited time.

* **Create an Engaging Learning Environment:** Encourage learning and interaction among intern teams while applying knowledge to practical scenarios.

---

## Competition Rules

* The competition consisted of **8 teams**, with **5 members** on each team.

* Two teams competed directly against each other in each match.

* Each match consisted of **10 questions**, ranging from easy to difficult.

* The team with the higher total score advanced to the next round.

* If both teams had the same score after 10 questions, a **Sudden Death question** was used. The team that answered correctly first won the match.

### Special Abilities

#### 1. Minimum Risk
* Each team could use this ability **once**.
* An incorrect answer would result in **no point deduction**.
* A correct answer would earn only **50% of the question's points**.

#### 2. Star of Hope
* Each team could use this ability **once**.
* A correct answer would earn **double points**.
* An incorrect answer would result in a **double point deduction**.

The questions covered common AWS services such as:

* Amazon EC2
* Amazon S3
* Amazon VPC
* AWS Lambda
* Amazon API Gateway
* Amazon DynamoDB
* Amazon CloudFront
* Amazon RDS
* AWS IAM
* Amazon Route 53

---

### 2. System Architecture Design

Participants had to analyze the requirements of each problem and select an appropriate architecture, including:

* Designing scalable systems
* Ensuring High Availability
* Optimizing operational costs
* Securing the system
* Providing Fault Tolerance

---

### 3. Real-World Scenarios

Many questions were based on practical implementation scenarios such as:

* Designing a Web Application architecture on AWS.
* Selecting an appropriate data storage service.
* Designing a Serverless system.
* Analyzing Disaster Recovery solutions.
* Optimizing AWS costs.
* Selecting services that meet performance and security requirements.

---

### 4. Competition Strategy

In addition to technical knowledge, teams had to determine the appropriate time to use the two special abilities to maximize their scores.

Balancing the confidence level of an answer against its potential point value introduced a strategic element that made the competition more engaging and competitive.

## Application to the Internship Project

The knowledge gained from the event can be directly applied to the project:

**"Serverless & Spot Instance Backend Architecture for Live-Service Games on AWS"**

Specifically:

* Select appropriate AWS services for each system component.
* Design an architecture that can scale as the number of players increases.
* Optimize costs by combining Serverless services and Amazon EC2 Spot Instances.
* Apply High Availability and Fault Tolerance principles when designing the Game Backend.
* Build the system according to the recommendations of the AWS Well-Architected Framework.

* Teamwork and rapid decision-making skills play an important role when solving system architecture problems.
* The event reinforced the knowledge acquired during the internship program and established a foundation for applying it to the Multiplayer Game Backend project on AWS.
